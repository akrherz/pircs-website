Format of Hadley Centre Boundary Data

Naming format:   EEEEES.pCYYDHM.pp
-------------

EEEEE : Experiment: aaxne = control, aaxnf = increasing CO2
S     : Submodel identifier: a = atmosphere, o = ocean
C     : Data stream
YY    : Year, from base year 1800 \
M     : Month			  | Valid at the end of the last field
D     : Day 			  |   (chronologically) in the file
H     : Hour (0 = 0Z)		  /
The 'p' and '.pp' both signify that the file contains pp fields.

Codes for data stream (C)
-------------------------
d	File contains one year of 5 day means of:
	00101  POTENTIAL TEMPERATURE (OCEAN)  DEG.C
	00146  AICE : ICE CONCENTRATION
	00147  HICE: MEAN ICE DEPTH OVER GRIDBOX  M

j	File contains 10 days of 6 hourly instantaneous fields of:
	00001  PSTAR AFTER TIMESTEP
	00002  U COMPNT OF WIND AFTER TIMESTEP
	00003  V COMPNT OF WIND AFTER TIMESTEP
	00004  THETA AFTER TIMESTEP
	00010  SPECIFIC HUMIDITY AFTER TIMESTEP

-------------------

Example:   aaxnfa.phn9ab0.pp
-------
Experiment aaxnf (increasing CO2), atmosphere submodel.
Final time of data in file:
   year       2039 = 1800 + 23*10 + 9
   month      October
   day        11
   hour       0Z
Start time of data in file:
   The data is on stream j => start time is 10 days earlier at 0Z 01/10/2039 

Directory naming
----------------
Each directory name takes the part of the file name common to all files inside
it. E.g. directory aaxnea.pho7 contains files with names aaxnea.pho7*.pp.
